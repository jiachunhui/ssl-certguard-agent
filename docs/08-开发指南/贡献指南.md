 # 贡献指南

 > 欢迎参与 CertGuard Agent 开源项目！

 感谢您有兴趣为 CertGuard Agent 做出贡献。无论是 Bug 修复、新功能、文档改进还是其他形式的贡献，我们都非常欢迎。

 ## 行为准则

 本项目遵循开源社区的行为准则。请保持：

- **尊重** — 尊重其他贡献者的观点和时间
- **包容** — 欢迎不同背景的贡献者
- **协作** — 以建设性态度沟通和解决问题

 ## 贡献方式

 ### 1. 报告 Bug

 发现 Bug 时，请在 GitHub Issues 提交：

- 使用 **Bug Report** 模板
- 清晰描述复现步骤
- 提供 Agent 版本、操作系统、Web 服务类型
- 附上相关日志（注意隐去敏感信息）
- 如果可以，提供复现的最小化示例

 > ![截图] 此处占位：GitHub Issues 提交 Bug 页面的截图

 ### 2. 提交功能建议

 在 GitHub Issues 中使用 **Feature Request** 模板：

- 清晰描述使用场景和需求
- 说明期望的行为
- 如果可以，描述实现思路

 ### 3. 提交代码

 #### 分支策略

 - `main` 分支：稳定发布版本
 - `develop` 分支：开发主分支
 - `feature/xxx`：功能分支
 - `fix/xxx`：修复分支

 #### 开发流程

 1. Fork 本仓库
 2. 创建功能分支：`git checkout -b feature/my-feature`
 3. 编写代码并添加测试
 4. 确保构建通过：`dotnet build`
 5. 提交 Pull Request 到 `develop` 分支

 #### 编码规范

 - 遵循 C# 编码规范（使用 `dotnet format` 检查）
 - 使用 .NET 8 的 `file` 作用域命名空间（已有文件的 file static 等保持风格一致）
 - 添加必要的 XML 注释（特别是公开 API）
 - 使用 `var` 当类型从右侧明显可见时
 - 遵循现有代码的命名风格和项目结构

 #### Commit 规范

 建议使用以下前缀：

| 前缀 | 说明 |
|------|------|
| `feat:` | 新功能 |
| `fix:` | Bug 修复 |
| `docs:` | 文档更新 |
| `refactor:` | 代码重构 |
| `test:` | 测试相关 |
| `chore:` | 构建/工具/依赖更新 |

 示例：

 ```
feat: 支持 Caddy 证书部署
fix: IIS 多域名绑定 SNI 标志设置失败
docs: 更新快速入门中的安装命令
 ```

 ### 4. 改进文档

 文档同样重要！包括：

- 修正拼写错误或语法问题
- 补充缺少的使用场景
- 添加更多示例
- 翻译文档（未来计划支持英文）

 文档位于 `docs/` 目录，使用 Markdown 格式。

 ## Pull Request 流程

 1. 确保 PR 描述清晰地说明改动内容和原因
 2. 关联相关的 Issue（如果有）
 3. 确保所有 CI 检查通过
 4. 等待至少一位维护者 Review
 5. 根据反馈进行修改

 ## 开发环境

 ### 推荐工具

- **IDE**: Visual Studio 2022 / JetBrains Rider / VS Code
- **SDK**: .NET 8 SDK
- **源码管理**: Git
- **终端**: Windows Terminal（Windows）/ 系统终端（Linux）

 ### 本地调试技巧

 1. 启动本地测试平台（或使用 `--server` 指定开发环境地址）
 2. 使用 `--insecure` 跳过自签证书验证
 3. 使用 `--heartbeat 10` 缩短心跳间隔加快调试

 ```bash
 dotnet run -- --server http://localhost:5003 --token ct_reg_test --heartbeat 10 --insecure
 ```
