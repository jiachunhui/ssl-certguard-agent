 # 更新与卸载

 > 如何更新 Agent 版本以及彻底卸载 Agent

 ## 更新 Agent

 ### 自动更新（推荐）

 Agent 会自动通过心跳检测平台端的最新版本。当发现版本不一致时，Agent 会自动执行以下步骤：

 1. 检测本机架构（linux-x64 / linux-arm64 / win-x64）
 2. 从平台下载对应架构的更新包
 3. 备份当前版本
 4. 解压并替换文件
 5. 退出主进程，由更新脚本自动重启服务

 整个过程无需人工介入。

 ### 手动更新

 如果希望手动控制更新时机：

 ```bash
certguard-agent --update https://agent.topssl.cn/download/certguard-agent-linux-x64.zip
```

 Agent 会下载更新包、备份旧版本、替换文件，然后退出。更新脚本会重新启动服务。

 更新完成后，可在日志中查看新版本号确认更新成功。

 ### Windows 更新注意

 Windows 下更新脚本通过 `sc start` 重新启动服务。确保服务名正确：

 ```powershell
 sc query TopSSLCertGuardAgent
 ```

 ## 卸载 Agent

 ### 标准卸载

 ```bash
certguard-agent --uninstall
```

 卸载过程执行以下步骤：

 1. **停止并删除服务** — 停止运行中的 Agent 服务，并从系统中移除
 2. **清理 PATH 环境变量**（仅 Windows）— 移除安装目录的系统 PATH
 3. **删除数据目录** — 删除日志、配置、临时证书等所有数据
 4. **删除安装目录** — 删除 Agent 二进制文件和相关文件

 ### 保留数据卸载

 如果希望保留配置和日志数据：

 ```bash
certguard-agent --uninstall --keep-data
```

 数据目录会被保留，重新安装时可复用已有配置。

 ### Linux 卸载后清理

 如果使用 systemd 安装，卸载后还需要清理服务文件：

 ```bash
 rm -f /etc/systemd/system/topssl-certguard-agent.service
 systemctl daemon-reload
 ```

 ### Windows 删除残留文件

 标准卸载会自动清理。如果手动删除，请确认停止并删除了服务：

 ```powershell
 sc.exe stop TopSSLCertGuardAgent
 sc.exe delete TopSSLCertGuardAgent
 ```

 然后删除安装目录和数据目录。

 > 重新安装时，使用新的注册令牌即可重新注册。
