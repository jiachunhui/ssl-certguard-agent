 # 配置文件说明

 > agent.json 配置文件详解

 ## 配置文件位置

 Agent 启动时会按以下顺序查找 `agent.json`（找到即停）：

 | 平台 | 查找顺序 |
|------|----------|
| Windows | 1. 可执行文件同目录 2. `%ProgramData%\CertGuard\` 3. 当前工作目录 |
| Linux | 1. 可执行文件同目录 2. `/etc/certguard/` 3. 数据目录 4. 当前工作目录 |

 注册成功后，配置文件会自动保存到首次找到的路径（或数据目录）。

 ## 配置项详解

 ```json
 {
   "api_base_url": "http://localhost:5003",
   "agent_id": "agt_xxxxxxxxxxxx",
   "agent_secret": "sk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
   "heartbeat_sec": 60,
   "data_dir": "/var/lib/TopSSL-CertGuard-Agent"
 }
 ```

 | 字段 | 类型 | 说明 |
|------|------|------|
| `api_base_url` | string | 平台 API 地址，结尾不含 `/` |
| `agent_id` | string | Agent 唯一标识，注册时由平台分配 |
| `agent_secret` | string | HMAC-SHA256 签名密钥，注册时由平台分配 |
| `heartbeat_sec` | int | 心跳间隔（秒），范围 10-3600 |
| `data_dir` | string | 运行时数据目录，用于存储日志、临时证书文件等 |

 ## 手动编辑注意事项

 1. **编辑前请先停止服务**：`systemctl stop topssl-certguard-agent`
 2. `agent_secret` 是敏感字段，Linux 上文件权限为 `600`
 3. 修改 `api_base_url` 后需要确保新地址可访问
 4. 编辑完成后启动服务：`systemctl start topssl-certguard-agent`

 ## 密钥轮换

 建议定期轮换 HMAC 密钥：

 1. 在 TOPSSL.CN 控制台生成新的密钥
 2. 在服务器上执行：`certguard-agent --update-secret <新密钥>`
 3. 重启 Agent 服务使新密钥生效
