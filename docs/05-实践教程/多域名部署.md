 # 多域名部署教程

 > 使用通配符证书或多域名证书管理多个站点

 ## 场景说明

 一个证书可能同时覆盖多个域名，例如：

- 通配符证书：`*.example.com` 覆盖 `www.example.com`、`api.example.com`、`blog.example.com` 等
- 多域名证书（SAN）：同时覆盖 `example.com`、`example.net`、`example.org`

 Agent 完全支持上述场景，会自动将证书部署到所有匹配的站点。

 ## 工作原理

 当 Agent 收到包含多个域名的 `deploy_cert` 任务时：

 ### Nginx / Apache

 Agent 将证书和密钥写入以**主域名**命名的目录中，所有域名共享同一份证书文件。您需要在 Nginx 或 Apache 配置中为每个域名正确引用该证书路径。

 ```
/etc/nginx/ssl/example.com/
├── fullchain.pem
└── privkey.pem
```

 ### IIS

 Agent 会扫描所有站点，为每个声明的域名查找匹配的站点绑定，然后为每个匹配的站点添加 HTTPS 绑定并分配证书。

 Agent 自动处理：

- 查找所有 IIS 站点中匹配的域名绑定
- 为每个匹配绑定添加 HTTPS
- 将证书指纹分配给每个 HTTPS 绑定
- 启用 SNI 支持

 ## 配置示例

 ### Nginx 多站点配置

 对于多个子域名共享一个通配符证书：

 ```nginx
# www.example.com
 server {
     listen 443 ssl;
     server_name www.example.com;
     ssl_certificate     /etc/nginx/ssl/example.com/fullchain.pem;
     ssl_certificate_key /etc/nginx/ssl/example.com/privkey.pem;
     ...
 }

# api.example.com
 server {
     listen 443 ssl;
     server_name api.example.com;
     ssl_certificate     /etc/nginx/ssl/example.com/fullchain.pem;
     ssl_certificate_key /etc/nginx/ssl/example.com/privkey.pem;
     ...
 }
 ```

 ### Apache 多站点配置

 ```apache
 # www.example.com
 <VirtualHost *:443>
     ServerName www.example.com
     SSLCertificateFile      /etc/apache2/ssl/example.com/certificate.crt
     SSLCertificateKeyFile   /etc/apache2/ssl/example.com/private.key
     ...
 </VirtualHost>

# api.example.com
 <VirtualHost *:443>
     ServerName api.example.com
     SSLCertificateFile      /etc/apache2/ssl/example.com/certificate.crt
     SSLCertificateKeyFile   /etc/apache2/ssl/example.com/private.key
     ...
 </VirtualHost>
 ```

 ### IIS 多域名场景

 如果 IIS 上有多个站点绑定不同的域名：

 | 站点 | HTTP 绑定 | Agent 自动添加 |
|------|-----------|---------------|
| www.example.com | `*:80:www.example.com` | `*:443:www.example.com` + 证书 |
| api.example.com | `*:80:api.example.com` | `*:443:api.example.com` + 证书 |

 如果通配符证书覆盖 `*.example.com`，Agent 会在每个站点上独立完成 HTTPS 绑定。

 ## 验证多域名部署

 使用 OpenSSL 验证证书是否覆盖所有域名：

 ```bash
 openssl s_client -connect www.example.com:443 -servername www.example.com </dev/null 2>/dev/null | openssl x509 -noout -text | grep -A1 "Subject Alternative Name"
 ```

 输出应列出证书覆盖的所有域名。

 ## 注意事项

- 通配符证书在 Agent 中被视为多域名场景处理
- 所有域名共享同一份证书文件和密钥
- IIS 场景下，Agent 会为每个域名分别创建绑定
- 确保 Nginx / Apache 配置中每个 server block 正确指定了证书路径
