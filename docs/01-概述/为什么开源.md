 # 为什么开源

 > 让 SSL 证书管理的每一个环节都经得起审视

 ## 我们为什么选择开源

 ### 1. 透明建立信任

 证书管理涉及服务器的核心安全。作为一个管理私钥和证书的工具，**闭源意味着用户必须无条件信任我们**——这对安全产品来说是不够的。

 通过开源，任何人都可以：

- 审查代码，确认私钥确实不出服务器
- 验证签名算法和通信协议的设计是否合理
- 检查是否存在后门、数据泄露或其他安全风险
- 自行编译二进制文件，杜绝供应链投毒

 ### 2. 安全需要众测

 安全不是一个人或一个团队能独自保证的。开源让全球的安全研究者和工程师可以：

- 报告潜在漏洞
- 提出安全加固建议
- 贡献安全测试用例
- 帮助我们发现那些内部测试覆盖不到的边界情况

 ### 3. 用户有自主权

 开源意味着您不会被锁定：

- 您可以 Fork 项目进行定制化修改
- 即使 TOPSSL.CN 平台停止服务，Agent 代码依然可用
- 您可以审计每个版本的变更，决定是否升级
- 您可以在封闭网络环境中自行编译部署

 ### 4. 社区驱动进步

 开源社区的反馈是产品进化的最佳动力。我们欢迎：

- 功能提议和场景分享
- Bug 报告和修复
- 文档改进和翻译
- Web 服务支持扩展（如 Caddy、Tomcat 等）

 ## 我们的承诺

| 承诺 | 说明 |
|------|------|
| 代码全量开源 | 包括核心通信层、部署逻辑、所有模块 |
| 构建可复现 | 提供明确的构建指南，任何人都可自行编译 |
| 漏洞公开响应 | 安全问题和修复通过 GitHub Issues / Security Advisory 公开 |
| 许可证稳定 | Apache 2.0 许可证，永久有效 |

 ## 许可证

 本项目采用 **Apache License 2.0** 开源许可证。

 这意味着您可以：

- ✅ 商业使用 — 在商业产品中集成或使用
- ✅ 修改 — 修改源码以满足您的需求
- ✅ 分发 — 重新分发原始版或修改版
- ✅ 专利授权 — 许可证包含明确的专利授权条款

 需要您做到的：

- 📄 保留原始版权声明
- 📄 修改的文件需附带修改说明

 ## 与 TOPSSL.CN 平台的关系

 CertGuard Agent 作为开源组件，与 TOPSSL.CN 商业平台配合使用：

- **Agent 开源** — 守护进程代码完全开源，Apache 2.0 许可证
- **平台闭源** — TOPSSL.CN 证书管理平台为商业服务
- **解耦设计** — Agent 可对接任何兼容 API 的后端，不锁定在特定平台

 > 我们的目标是：让 SSL 证书管理变得安全、自动、透明。开源是实现这一目标的基础，而不是附加项。
