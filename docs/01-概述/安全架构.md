 # 安全架构

 > 从设计上保障私钥和通信安全

 ## 设计原则

 CertGuard Agent 的安全设计遵循以下核心原则：

 1. **最小权限** — Agent 只做一件事：部署证书，不做多余操作
 2. **私钥本地化** — 私钥始终在用户服务器上生成和存储
 3. **通信加密与认证** — 所有 API 请求携带 HMAC-SHA256 签名
 4. **防重放** — 每个请求携带唯一 nonce 和时间戳
 5. **可审计** — 完整日志记录，所有操作可追溯

 ## 通信安全

 ### HMAC-SHA256 签名机制

 Agent 与平台之间的每次 API 请求都经过签名认证：

 ```
 签名字符串 = HTTP方法 + "|" + 请求路径 + "|" + nonce + "|" + 时间戳
 签名 = HMAC-SHA256(secret, 签名字符串)
 ```

 请求头携带：

| 请求头 | 说明 |
|--------|------|
| `X-Agent-Id` | Agent 唯一标识 |
| `X-Agent-Timestamp` | Unix 时间戳（秒） |
| `X-Agent-Nonce` | 16 字节随机数（Hex 编码） |
| `X-Agent-Signature` | HMAC-SHA256 签名值 |

 ### 防重放攻击

- 每个请求的 **nonce 唯一**：16 字节加密随机数，几乎不可能重复
- **5 分钟时间窗口**：平台端验证时间戳是否在允许范围内
- 平台端对已使用的 nonce 进行记录，防止同一请求重复提交

 ### 注册安全

- 首次注册使用**一次性注册令牌**，注册成功后即失效
- 注册成功后平台返回 `agent_id` + `secret`，后续通信全部使用 HMAC 签名
- 注册令牌可通过 `--update-secret` 命令轮换

 ## 数据安全

 ### 私钥保护

- 私钥在服务器本地通过 PEM 文件存储
- Linux 上私钥文件权限自动设置为 `600`（仅属主可读写）
- 平台**不存储**任何私钥，只下发 CA 签名证书
- 私钥文件存储在 Agent 数据目录下，与其他文件隔离

 ### 配置文件保护

- `agent.json` 配置文件包含 `agent_id` 和 `agent_secret`
- Linux 上配置文件权限自动设置为 `600`
- 配置文件存储在安装目录或系统数据目录下

 ## 部署安全

 ### 证书写入

- Agent 将证书和密钥写入 Web 服务对应的配置目录
- 写入文件后执行配置检查（如 `nginx -t`），避免错误配置导致服务不可用
- 配置检查通过后才执行 Web 服务重载

 ### Web 服务重载

- Nginx：先 `nginx -t` 校验配置，再 `nginx -s reload` 热重载
- Apache：使用 `apache2ctl graceful` 优雅重载，不中断现有连接
- IIS：使用 `iisreset` 重置服务

 ## 自动更新安全

- Agent 通过心跳检测平台端的最新版本号
- 更新包通过 HTTPS 下载，确保传输安全
- 下载后校验解压，替换二进制前先备份旧版本
- 更新过程通过独立脚本执行，主进程先退出，更新脚本完成后重新启动服务

 ## 卸载安全

- 卸载时先停止服务，再从平台注销 Agent 身份
- 可选择保留或清除数据目录
- Windows 下自动清理 PATH 环境变量
- 使用防删除批处理脚本处理正在运行的可执行文件

 ## 审计日志

- Agent 所有关键操作记录到日志文件
- 日志按天滚动，保留最近 7 天
- 单文件大小上限 10MB，超限自动滚动
- 日志路径：`{DataDir}/logs/certguard-agent-yyyyMMdd.log`
