 # 概述

 > 安全、透明、可控的 SSL 证书自动化管理方案

 CertGuard Agent 是一个轻量级的后台守护进程，安装在您的 Web 服务器上，自动完成 SSL 证书的部署和续签。它配合 [TOPSSL.CN](https://topssl.cn) 平台使用，让证书管理从手动操作变为全自动化。

 ## 核心能力

- **自动发现** — Agent 启动后自动检测服务器上已安装的 Web 服务
- **自动部署** — 接收平台下发的证书任务，写入文件并重载 Web 服务
- **自动续签** — 周期性心跳检查，证书到期前自动续签
- **自动更新** — 检测到新版本时自动下载更新，无需人工介入

 ## 支持的服务

| Web 服务 | 支持平台 | 状态 |
|----------|----------|------|
| Nginx | Linux | ✅ 已支持，自动检测配置 |
| Apache | Linux | ✅ 已支持，自动检测配置 |
| IIS | Windows | ✅ 已支持，自动匹配站点绑定 |

 ## 核心原则：私钥不出服务器

 这是 CertGuard Agent 最重要的安全承诺。所有密钥对的生成和存储完全在您的服务器本地完成，TOPSSL.CN 平台仅下发经过 CA 签名的证书内容和任务指令。您的私钥始终掌握在您自己手中。

 ## 工作流程

 ```
 ┌─────────────┐     心跳 + 拉取任务     ┌──────────────┐
 │  TOPSSL.CN  │ ◄────────────────────── │   Agent      │
 │   Platform  │ ──────────────────────► │  (你的服务器) │
 └─────────────┘     上报结果             └──────┬───────┘
                                                 │
                                            ┌────▼────┐
                                            │  Nginx  │
                                            │  Apache │
                                            │   IIS   │
                                            └─────────┘
 ```

 1. Agent 启动后自动检测本地 Web 服务类型，上报环境信息
 2. 首次运行使用一次性令牌注册，之后全部使用 HMAC 签名通信
 3. 每 60 秒发送心跳，同时拉取待执行的部署任务
 4. 收到 `deploy_cert` 任务后：写入证书文件 → 重载 Web 服务 → 上报结果
